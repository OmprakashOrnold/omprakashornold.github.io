Java Introduction

-> Java is a programming language which is used to communicate with Computers

-> Java is free and open source programming language

-> Java developed by Sun Microsystem in 1991 ( Name was OAK )

-> In 1995 they renamed OAK language to JAVA language

-> In 2010 Sun Microsystem acquired by Oracle Corp..

Note: Now Java is under license of Oracle company

-> As per Sun Microsystem java is divided into 3 parts

  1. J2SE / JSE

  2. J2EE / JEE

  3. J2ME / JME

-> J2SE / JSE –> Java Standard Edition —> Stand Alone applications

-> Standalone applications will run only in one computer

Ex: Games, Notepad, Caclulator, MS Paint, OS, Eclipse IDE etc....

-> J2EE / JEE –> Java Enterprise Edition —> Web Applications

-> Any Application which can be accessed by Multiple users at a time it is called as Web Application

Ex: Gmail, Facebook, IRCTC, Linkdin, Naukri etc…..

-> J2ME / JME –> Java Micro Edition / Java Mobile Edition —> Mobile Applications

-> Any Application which we can install and use in our mobiles is called as Mobile Application

Ex: WhatsApp, Google Pay & Phone Pay etc….

Note: Now a days Java is mainley used for Web Application development.

Developing First Java Program

  1. We can write java programs in any text editor

    Ex: Notepad, Notepad++, Edit Plus, Eclipse IDE etc…

  2. We can compile and execute our java program in ‘Command Prompt’

  3. Java program should have .java extension for the files (Ex: Welcome.java, Demo.java, Hello.java etc…)

  4. When we compile java program it will generate .class file if source code is correct.

  5. We can compile java program using below command

    /> javac <filename.java>

  6. Once .class file is created then we can run our java program

  7. We can run java program using below command

    /> java <filename>

Note: When we run our java program, JVM will start and it will execute our program.

Translators

-> Translators are used to convert from one form to another form

-> In computer programming, we have 3 types of translators

  1. Interpreter

  2. Compiler

  3. Assembler

-> Interpreter will translate the program line by line. Interpreter Performance is very slow.

-> Compiler will translate all the lines of the program at a time. Compiler performance is fast.

-> Assembler is used to translate assembly languages code into machine code.

-> Java Slogan —-> WORA –> Write Once Run Anywhere

Note: C language is platform dependent

JVM Architecture

-> JVM stands for Java Virtual Machine

-> JVM will be available inside JRE

-> JVM is responsible to execute Java Programs

-> Java Program Execution means take the byte code and convert into Machine Code

-> To execute java program we need some memory that memory allocation & de-allocation will be taken care by JVM

-> JVM is responsible for Memory Management in java

JVM Components

  1. Class Loader : It is responsible to load .class file into JVM. After loading .class file, JVM will read byte code and it will take care of memory management & program execution.

  2. Method Area : It is used store our class code

  3. Heap area : It is used to store our Objects.

  4. 8 : It is used to store method execution data

  5. PC Register : It is used to maintain instructions execution order (PC stands for Program Counter)

  6. Native Stack : It is used to store non-java code execution details

  7. Native Libraries : These are used to execute Native Code (Non-Java Code)

  8. Native Interface : It acts as mediator between Execution Engine and Native Libraries. It will load Native Libraries into JVM

  9. Execution Engine : It is used to execute our program that means it Converts byte code into machine code.

Note: Execution Engine will use ‘Interpreter + JIT’ to perform program execution.

-> Interpreter will execute the program line by line. To improve performance Sun Microsystem added JIT in JVM.

-> JIT stands for Just in Time Compiler
